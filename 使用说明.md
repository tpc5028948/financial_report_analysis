# 企业信用评估模型 - 使用说明

## 问题解决：评估失败 "Failed to fetch"

### 原因分析
这个错误是因为**后端服务没有运行**，或者前后端端口不匹配。

### 已完成的修复

1. ✅ **统一端口**：前端和后端都使用 **8000** 端口
2. ✅ **CORS配置**：后端已配置跨域支持
3. ✅ **文件解析增强**：支持Excel/PDF智能解析
4. ✅ **财务指标计算**：ROE、ROA等自动计算

## 启动步骤

### 方法一：双击批处理文件（推荐）

1. 找到项目目录中的 `启动服务.bat`
2. 双击运行
3. 等待服务启动（看到 "Running on http://0.0.0.0:8000"）
4. 在浏览器中打开 `frontend/index.html`

### 方法二：命令行启动

1. 打开命令提示符（CMD）或PowerShell
2. 进入项目目录：
   ```
   cd C:\Users\Administrator\Documents\trae_projects\financial_report_analysis
   ```
3. 运行命令：
   ```
   C:\Users\Administrator\AppData\Local\Programs\Python\Python312\python.exe backend/api/app.py
   ```
4. 在浏览器中打开 `frontend/index.html`

## 验证服务是否正常

### 1. 检查后端服务
在浏览器访问：
```
http://localhost:8000/api/health
```

如果看到：
```json
{
  "status": "ok",
  "service": "企业信用评估模型"
}
```
说明后端服务运行正常。

### 2. 测试文件上传

- 使用示例数据或者上传财务报表Excel
- 点击"开始信用评估"
- 查看结果是否正常显示

## 常见问题

### Q: 端口被占用怎么办？
A: 修改 `backend/api/app.py` 和 `run_server.py` 中的 `port=8000` 为其他端口，同时修改前端HTML中的API地址。

### Q: 如何验证后端正在运行？
A: 打开浏览器访问 http://localhost:8000/api/health

### Q: 文件上传失败？
A: 确保后端服务已启动，检查浏览器控制台是否有错误信息。

### Q: 看不到财务数据？
A: 检查上传的Excel是否包含标准财务指标名称（如"营业收入"、"净利润"等）。
